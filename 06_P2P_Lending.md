# ТЗ: P2P Lending Protocol на Bitcoin

## Обзор проекта

Простой P2P лендинг на Bitcoin. Один пользователь даёт BTC в долг, другой берёт под залог (Ordinals NFT или BRC-20 токены). Фиксированная ставка, фиксированный срок. Без пулов ликвидности.

---

## Целевая аудитория

- Держатели BTC, желающие зарабатывать на процентах
- Владельцы NFT/BRC-20, нуждающиеся в ликвидности без продажи
- DeFi-пользователи, привыкшие к Aave/Compound

---

## Функциональные требования

### 1. Подключение кошелька

- Поддержка Unisat, Xverse
- Чтение BTC баланса, Ordinals, BRC-20

### 2. Создание предложения (Offer)

**Для лендера (кредитор):**
- Сумма к выдаче (в BTC/sats)
- Процентная ставка (% годовых)
- Срок займа (7/14/30/90 дней)
- Требуемый тип залога: Ordinals / BRC-20
- Минимальная стоимость залога (LTV ratio)

**Для заёмщика:**
- Запрос суммы
- Залог: выбор Ordinal или BRC-20 из кошелька
- Желаемая ставка и срок

### 3. Мэтчинг

- Маркетплейс открытых предложений
- Заёмщик выбирает предложение лендера (или наоборот)
- Кнопка "Accept" — создание займа
- При мэтчинге: залог блокируется, BTC отправляется заёмщику

### 4. Управление займом

- Дашборд активных займов
- Таймер до погашения
- Кнопка "Repay" — возврат BTC + проценты
- При погашении: залог возвращается заёмщику
- При дефолте: залог передаётся лендеру

### 5. Ликвидация

- Автоматическая при истечении срока без погашения
- Залог переходит лендеру
- Уведомление обеим сторонам
- Grace period: 24 часа после дедлайна

### 6. Маркетплейс предложений

- Список всех открытых предложений (лендеров и заёмщиков)
- Фильтры: по сумме, ставке, сроку, типу залога
- Сортировка по ставке, сумме

## Механика займа

```
1. Лендер создаёт предложение: 0.1 BTC, 15% APR, 30 дней
2. Заёмщик принимает, блокирует Ordinal NFT как залог
3. Escrow: залог → мультисиг-адрес (2-of-3: лендер, заёмщик, платформа)
4. BTC отправляется заёмщику

Через 30 дней:
  A) Заёмщик возвращает 0.1 BTC + 0.00123 BTC (проценты за 30 дней)
     → Залог разблокируется → возврат заёмщику
  B) Заёмщик не платит
     → Grace period 24 часа
     → Залог передаётся лендеру

Расчёт процентов: 
  interest = principal × (APR / 365) × days
  0.1 × (0.15 / 365) × 30 = 0.00123 BTC
```

## MVP-скоуп

1. Создание предложений от лендера
2. Принятие предложения заёмщиком с блокировкой залога
3. Отправка BTC заёмщику
4. Погашение займа
5. Ликвидация при дефолте

---

## Что НЕ входит в MVP

- Пулы ликвидности
- Переменные ставки
- Частичное погашение
- Вторичный рынок займов
- Автоматическая оценка залога
- Flash loans

## Риски и митигация

| Риск | Митигация |
|------|-----------|
| Волатильность залога | Overcollateralization (LTV 50-70%) |
| Escrow security | Мультисиг + аудит |
| NFT-залог сложно оценить | Floor price от Ordinals маркетплейсов |
| Заёмщик не платит | Автоматическая ликвидация + grace period |
